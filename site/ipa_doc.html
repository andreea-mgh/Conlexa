Wordshift Documentation

Wordshift is a tool for applying sound changes to words. It is enabled by default on dictionary entries and checks if a valid ruleset has been defined for the word's language.

To define a ruleset, go to the IPA Rulesets page and click on a language. The ruleset editor allows you to define sound changes using a simple syntax. Once you have defined a ruleset, it will be automatically applied to all words in that language as long as the syntax is valid.

Syntax

Sections

A ruleset is divided into sections, defined by a $ symbol at the beginning of the line.

$GROUP

This section defines symbols or groups of symbols that can be referenced in the rules. Each line defines a group in the format NAME:symbol1,symbol2,... where NAME is a capital letter. For example:

$GROUP
V:a,e,i,o,u
K:p,t,k

This defines a group V that represents the vowels and a group K that represents the voiceless stops.

$SUBST

This section defines the first stage of substitution in the ruleset. What is particular about this section is the fact that the substitutions will not overlap. It is meant for general letter/digraph to sound mapping. The syntax is also simpler than in the other substitution sections, with the only format accepted being pattern:replacement.

For example:

$SUBST
ei:e
e:ɛ
ch:k
sh:ʃ
th:θ

If this ruleset would be applied to a word like chein, it would be transformed to chen after the first rule but the second rule would not be able to transform it further into chɛn because the "ei" group has already been substituted, making this section perfect for simple sound mapping.

$POST

This section defines the second stage of substitution in the ruleset. The syntax is more complex than in the $SUBST section. The minimal syntax is pattern:replacement, but can be expanded with an optional parameter: pattern:replacement:context. The context part is optional and can be used to specify when the substitution should occur.

This section can also reference groups defined in the $GROUP section.

The context part is a string containing an underscore, representing the substitution. In this string you can also include the word boundary as %.

For example, if you have a group V:a,e,i,o,u in the $GROUP section, you can use it in the $POST section like this:

$POST
V:ɛ:V_
V:ɔ:_V

This would replace vowels with ɛ after another vowel and with ɔ before another vowel.

Keep in mind that these substitutions can be applied on top of each other.

Comments

You can add comments to your ruleset by starting a line with #. Comments are ignored by the ruleset processor and can be used to explain the purpose of certain rules or to temporarily disable them. Comments are stored on save.

The processor also ignores newlines and whitespace.
